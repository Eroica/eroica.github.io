I favor Cathedral-style development models, and providing pre-compiled binaries without installers and all dependencies included. SQLite is my role model in this regard. An application should be downloadable directly from the vendor without relying on system package managers. If I need a program or library, I download or compile from upstream sources, and put them into a directory managed by GNU Stow. This allows me to have different versions of the same library installed at the same time, without any name mangling, and which can be activated and deactivated with stow [-D]. I think the Filesystem Hierarchy Standard should be replaced by what GoboLinux does. Those who do not understand Objective-C are bound to reinvent it.
